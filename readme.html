Данные тайлов лежат в папке "tiles_data"

Внутри лежат папки - по одной на каждый тайл. Именование используется как id для каждого тайла, этот id передаётся на сервер в момент запроса информации для сожержимого плашки, так что желательно использовать максимально простые названия типа "01", "02" и т.п.

Внутри каждой папки лежит файл "data.json". Он сожержит основную информацию по тайлу. Его синтаксис:

{
  "tile": {
    "backgroundFilename": "строка",
    "backgroundColor": "Blue" | "Red" | "Green",
    "icons": [ Icon1, Icon, ... ],
    "caption": "строка",
    ["fontSize": "big" | "medium" | "small"]
  },
  "content": {
    "title": "строка"
  }
}
,

где 1Icon = "Metallurgy" | "Mech" | "Food" | "Oil" | "Medicine" | "Repair"

"fontSize" - единственный необязательный параметр. Его значение по умолчанию - "big"

Файлы бекграунда для параметра "backgroundFilename" хранятся в папке "images/tile_backgrounds"

Текст внутри плашки хранится внутри отдельного файла "text.txt" в той же папке.

Изображения хранятся в папке images. Названия файлов без расширения используются как подписи к ним внутри плашки.

При каждом запросе к серверу файлы (в частности, файл data.json) валидируются на корректность. Ошибки, если они есть, записываются в файл "validationErrors.txt".
Если ошибок нет - файл не создаётся.

Ошибки сервера, если таковые возникнут, записываются в файл "logs.txt"
